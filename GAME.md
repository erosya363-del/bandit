# БАНДИТ — описание игры

> Вертикальный браузерный платформер в духе Doodle Jump.  
> Герой — **Человек-Паук**. Поднимайтесь всё выше, собирайте бонусы, уклоняйтесь от ломающихся платформ и боритесь за место в топ‑10.

**Демо:** https://erosya363-del.github.io/bandit/  
**Репозиторий:** https://github.com/erosya363-del/bandit  
**Платформа:** веб (десктоп + мобильные браузеры), без установки

---

## 1. Суть игры

Игрок управляет Человеком-Пауком, который прыгает по платформам в бесконечно растущем вертикальном уровне. Цель — набрать максимальный счёт, поднимаясь как можно выше, не упав вниз.

Перед стартом нужно ввести **новое имя** — результат попадает в локальный рейтинг топ‑10.

Ключевые особенности:

- **Ручной прыжок** — не автоматический; до **2 прыжков** в воздухе.
- **Свайп-управление** на мобильных — движение только при перетаскивании пальца.
- **Единый рейтинг** — одинаковый на всех экранах и вкладках браузера.

## 2. Жанр и игровой цикл

| Параметр | Значение |
|----------|----------|
| Жанр | Casual / arcade, vertical platformer |
| Перспектива | 2D, вид сбоку |
| Сессия | 1–5 минут, бесконечный уровень до проигрыша |
| Цикл | Прыжок → приземление → движение → сбор бонуса → выше → сложнее |

**Игровой цикл:**

1. Игрок **стоит** на платформе (ступни на поверхности).
2. Нажимает прыжок — Человек-Паук **поднимает руки**, стреляет паутиной.
3. Приземляется на следующую платформу — получает очки за высоту.
4. Собирает бонусы — растёт комбо.
5. Камера следует вверх; новые платформы генерируются над игроком.
6. Падение за экран = −1 жизнь (всего 3). После 0 жизней — Game Over.
7. Счёт сохраняется в рейтинг, показывается место.

## 3. Управление

### Десктоп

| Действие | Клавиши |
|----------|---------|
| Влево | `←` или `A` |
| Вправo | `→` или `D` |
| Прыжок | `Пробел`, `↑` или `W` |

### Мобильные устройства

| Действие | Жест / элемент |
|----------|----------------|
| Ходьба | **Водить пальцем** влево или вправо по экрану |
| Стоять | Положить палец — персонаж **не двигается** |
| Прыжок | **Короткий тап** без движения (< 280 мс) |
| Альтернатива | Кнопки **◀ ▶ ⇧** внизу экрана |

**Правила свайп-управления:**

- Движение идёт **в сторону перетаскивания**, а не к точке касания.
- Убрали палец — движение **прекращается**.
- Снова коснулись экрана — неважно где; персонаж стоит, пока не проведёте пальцем.
- Палец неподвижен на экране — персонаж **не двигается**.

**Важно:** прыжок **не происходит автоматически** при приземлении.

## 4. Механики

### 4.1. Прыжки

- Максимум **2 прыжка** до следующего приземления.
- Счётчик в HUD: `КОМБО·ПРЫЖ → x1 · ⬆2`.
- На земле доступны 2 прыжка; после первого в воздухе остаётся 1.
- При прыжке — анимация **руки вверх** и белые **нити паутины**.

### 4.2. Платформы

| Тип | Поведение | Визуал |
|-----|-----------|--------|
| **Обычная** | Статичная | Цветной прямоугольник с закруглёнными краями |
| **Ломающаяся** | После приземления — предупреждение ~2 сек, затем исчезает | Мигает красным, пунктир, таймер «2с…1с» |
| **Движущаяся** | Ездит влево-вправo | Голубое свечение |

- Платформы **сжимаются** (squash) при приземлении.
- Если игрок стоит на ломающейся платформе в момент обрушения — падает.

### 4.3. Прогрессия сложности

Чем выше поднимаетесь:

- платформы **уже** и **реже**;
- чаще **ломающиеся** и **движущиеся**;
- реже бонусы.

### 4.4. Бонусы

| Предмет | Очки (× комбо) | Внешний вид |
|---------|----------------|-------------|
| Звезда | 50 | Золотая звезда |
| Подушка | 25 | Розовый прямоугольник |
| Игрушка | 100 | Бирюзовый шар |

- Покачивание (bob), частицы и звук при сборе.
- При комбо > 2 — всплывающий текст `x3!`, `x4!` и т.д.

### 4.5. Комбо

- Каждый бонус увеличивает множитель (макс. **×10**).
- Сброс, если ~3 секунды не собирать бонусы.
- Комбо умножает очки за бонусы и за подъём.

### 4.6. Жизни

- Старт с **3 жизнями** (♥♥♥).
- Падение за экран = −1 жизнь, respawn на ближайшей платформе.
- Краткая неуязвимость после respawn (мигание).
- 0 жизней → Game Over.

### 4.7. Камера

- Плавно следует, когда игрок выше 38% экрана.
- Parallax-фон (город, звёзды).

## 5. Система очков

| Источник | Формула |
|----------|---------|
| Подъём вверх | `floor(пройденная_высота × 0.1) × комбо` |
| Звезда | `50 × комбо` |
| Подушка | `25 × комбо` |
| Игрушка | `100 × комбо` |

Личный рекорд сохраняется отдельно и показывается на Game Over.

## 6. Рейтинг (leaderboard)

### Правила

- Перед игрой **обязательно** ввести имя: 2–20 символов (буквы кириллицы/латиницы, цифры).
- **Каждая новая игра — новое имя.** Повторить имя прошлой сессии нельзя — нужно ввести другое.
- Сохраняется **топ‑10** лучших результатов в `localStorage` браузера.
- **Одно имя = одна запись** в таблице (при повторном попадании обновляется лучший счёт).
- Каждый счёт **подписан** хешем — подделка через консоль не проходит проверку.
- **Единый рейтинг** на всех экранах: старт, HUD, Game Over, кнопка «🏆 Рекорды».
- Синхронизация между вкладками и при возврате на страницу.
- Автоматический перенос данных из старого ключа `bandit_leaderboard` → `bandit_leaderboard_v4`.

### Где показывается

| Место | Что видно |
|-------|-----------|
| HUD во время игры | Live-рейтинг (#1, #2, …) |
| Стартовый экран | Топ‑10 по кнопке «🏆 Рекорды» |
| Game Over | Финальное место (🥇🥈🥉 / #N) + таблица |

### Ограничения

- Рейтинг **локальный** (в браузере на устройстве), не глобальный для всех игроков.
- При очистке данных браузера рейтинг сбрасывается.
- Разные URL (localhost / GitHub Pages) — разные хранилища браузера.

## 8. Персонаж

### Человек-Паук

- Стилизованный 2D-силуэт: красная маска с белыми глазами, синий торс с паутиной и символом паука, красные перчатки и ботинки.
- **Стоит** на платформе (ступни на поверхности, прямые ноги).
- Анимации: ходьба, прыжок с **руками вверх** и паутиной, поворот по направлению.
- При respawn — мигание (неуязвимость).

### Кастомизация: загрузка лица

- На экране старта можно загрузить **фото лица**.
- Белый фон удаляется автоматически.
- Лицо вставляется в круглую маску вместо стандартной головы.
- Кнопка «✕ Убрать» — вернуть стандартную маску.

## 11. Техническая информация

| Параметр | Значение |
|----------|----------|
| Стек | Vanilla JS, HTML5 Canvas, CSS3 |
| Game loop | Fixed timestep 60 Hz |
| Рендер | Canvas 2D, DPR-aware |
| Звук | Web Audio API |
| Хранение | localStorage (`bandit_leaderboard_v4`, `bandit_best`, `bandit_last_player`) |
| Деплой | GitHub Pages |

### Структура кода

```
js/config.js    — константы, палитра, сложность
js/math.js      — векторы, коллизии, утилиты
js/storage.js   — рейтинг, подпись счёта, миграция
js/input.js     — клавиатура, свайп, кнопки
js/entities.js  — игрок, платформы, бонусы
js/systems.js   — частицы, звук, фон, спавнер
js/game.js      — ядро, state machine, loop
index.html      — разметка экранов
style.css       — UI
GAME.md         — этот документ
```
