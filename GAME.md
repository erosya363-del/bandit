# БАНДИТ — описание игры

> Вертикальный браузерный платформер в духе Doodle Jump.  
> Герой — **Человек-Паук**. Игрок поднимается всё выше, собирает бонусы, уклоняется от ломающихся платформ и соревнуется за место в топ‑10.

**Демо:** https://erosya363-del.github.io/bandit/  
**Репозиторий:** https://github.com/erosya363-del/bandit  
**Платформа:** веб (десктоп + мобильные браузеры), без установки

---

## 1. Суть игры (elevator pitch)

Игрок управляет Человеком-Пауком, который прыгает по платформам в бесконечно растущем вертикальном уровне. Цель — набрать максимальный счёт, поднимаясь как можно выше, не упав вниз. Игра соревновательная: перед стартом нужно ввести имя, результат попадает в локальный рейтинг топ‑10.

Ключевая особенность — **ручной прыжок** (не автоматический): игрок сам решает, когда прыгать, и может сделать **до двух прыжков подряд в воздухе**. Это добавляет контроль и риск, в отличие от классического auto-bounce.

---

## 2. Жанр и игровой цикл

| Параметр | Значение |
|----------|----------|
| Жанр | Casual / arcade, vertical platformer |
| Перспектива | 2D, вид сбоку |
| Сессия | 1–5 минут, бесконечный уровень до проигрыша |
| Цикл | Прыжок → приземление → движение → сбор бонуса → выше → сложнее |

**Игровой цикл (loop):**

1. Игрок стоит на платформе.
2. Нажимает прыжок (или делает второй прыжок в воздухе).
3. Приземляется на следующую платформу — получает очки за высоту.
4. Собирает бонусы на платформах — умножает счёт комбо.
5. Камера следует вверх; новые платформы генерируются над игроком.
6. При падении за экран теряет жизнь (3 жизни). После 0 жизней — Game Over.
7. Счёт сохраняется в рейтинг, показывается место.

## 3. Управление

### Десктоп

| Действие | Клавиши |
|----------|---------|--------|
| Влево | `←` или `A` |
| Вправо | `→` или `D` |
| Прыжок | `Пробел`, `↑` или `W` |

### Мобильные устройства

| Действие | Жест / элемент |
|----------|----------------|
| Ходьба | Водить пальцем по экрану (персонаж идёт к точке касания) |
| Прыжок | Короткий тап по экрану (без движения) |
| Альтернатива | Кнопки **◀ ▶ ⇧** внизу экрана |

**Важно:** прыжок **не происходит автоматически** при приземлении. Игрок должен нажать прыжок сам.

## 4. Механики

### 4.1. Прыжки

- Максимум **2 прыжка** до следующего приземления.
- Счётчик оставшихся прыжков отображается в HUD: `КОМБО·ПРЫЖ → x1 · ⬆2`.
- На земле всегда доступны 2 прыжка; после первого в воздухе остаётся 1.

### 4.2. Платформы

| Тип | Поведение | Визуал |
|-----|-----------|--------|
| **Обычная** | Статичная, не ломается | Цветной прямоугольник с закруглёнными краями |
| **Ломающаяся** | После первого приземления — предупреждение ~2 сек, затем исчезает | Мигает красным, пунктирная обводка, таймер «2с…1с» |
| **Движущаяся** | Ездит влево-вправо по экрану | Голубое свечение (glow) |

- Платформы **сжимаются** (squash) при приземлении — лёгкий juice-эффект.
- Если игрок стоит на ломающейся платформе в момент обрушения — падает.

### 4.3. Прогрессия сложности по высоте

Чем выше игрок поднимается:

- платформы становятся **уже** и **реже**;
- чаще появляются **ломающиеся** и **движущиеся** платформы;
- реже спавнятся бонусы.

Конкретные параметры зависят от выбранной сложности (см. раздел 7).

### 4.4. Бонусы (collectibles)

| Предмет | Очки (× комбо) | Внешний вид |
|---------|----------------|-------------|
| Звезда | 50 | Золотая звезда, вращается |
| Подушка | 25 | Розовый прямоугольник |
| Игрушка | 100 | Бирюзовый шар |

- Предметы **покачиваются** (bob-анимация) над платформой.
- При сборе — частицы, звук, рост комбо.
- При комбо > 2 — всплывающий текст `x3!`, `x4!` и т.д.

### 4.5. Комбо

- Каждый собранный бонус увеличивает множитель комбо (макс. **×10**).
- Комбо **сбрасывается**, если ~3 секунды не собирать бонусы.
- Комбо умножает: очки за бонусы и очки за подъём вверх.

### 4.6. Жизни

- Старт с **3 жизнями** (♥♥♥).
- Падение за нижнюю границу экрана = −1 жизнь, respawn на ближайшей платформе.
- Краткая неуязвимость после respawn (мигание персонажа).
- 0 жизней → Game Over.

### 4.7. Камера

- Плавно следует за игроком, когда он поднимается выше 38% экрана.
- Фон (город, звёзды) движется с parallax-эффектом.

---

## 5. Система очков

| Источник | Формула |
|----------|---------|
| Подъём вверх | `floor(пройденная_высота × 0.1) × комбо` за каждый шаг |
| Звезда | `50 × комбо` |
| Подушка | `25 × комбо` |
| Игрушка | `100 × комбо` |

Личный рекорд сохраняется отдельно и показывается на экране Game Over.

## 6. Рейтинг (leaderboard)

### Правила

- Перед игрой **обязательно** ввести имя: 2–20 символов (буквы кириллицы/латиницы, цифры).
- Сохраняется **топ‑10** лучших результатов (localStorage браузера).
- Каждый счёт **подписан** криптографическим хешем — подделка через консоль не проходит проверку.
- Во время игры в HUD показывается **live-рейтинг** — какое место занял бы текущий счёт.
- На Game Over — финальное место (🥇🥈🥉 или #N) и таблица топ‑10.

### Ограничения

- Рейтинг **локальный** (на устройстве / в браузере), не общий для всех игроков в интернете.
- При очистке данных браузера рейтинг сбрасывается.

## 7. Уровни сложности

| Параметр | Легко | Средне | Сложно |
|----------|-------|--------|--------|
| Расстояние между платформами | широкое (155 px база) | среднее (125 px) | узкое (95 px) |
| Сила прыжка | −16.5 | −15 | −14 |
| Рост сложности с высотой | медленный | средний | быстрый |

Выбор — на экране старта, кнопки «Легко / Средне / Сложно». По умолчанию — **Средне**.

## 8. Персонаж

### Человек-Паук (базовый)

- Стилизованный 2D-силуэт: красная маска с белыми «глазами», синий торс, красный паук на груди, синие/красные конечности.
- Анимации: ходьба (покачивание), прыжок (белые «нити» паутины), поворот по направлению движения.
- При respawn — мигание (неуязвимость).

### Кастомизация: загрузка лица

- На экране старта можно загрузить **фото лица**.
- **Белый фон удаляется автоматически** (прозрачность).
- Лицо вставляется в круглую маску вместо стандартной «головы» Spider-Man.
- Можно убрать (кнопка «✕ Убрать»).

## 9. Экраны и интерес UI

### 9.1. Экран старта (Start Screen)

| Элемент | Назначение |
|---------|------------|
| Логотип 🕷️ + «БАНДИТ» | Брендинг |
| Подзаголовок | «Человек-Паук · Ночной город · Рейтинг» |
| Поле имени | Ввод никнейма (обязательно) |
| Выбор сложности | 3 кнопки |
| Загрузка аватара | Превью + кнопка «Загрузить лицо» |
| Топ‑10 | Скрыт по умолчанию, открывается кнопкой «🏆 Рекорды» |
| Подсказка управления | «👆 Водите — ходьба · Тап / ⇧ / Пробел — прыжок (×2)» |
| «НАЧАТЬ ИГРУ» | Старт |

Экран **прокручивается** на маленьких экранах (overflow-y: auto).

### 9.2. Игровой HUD (поверх canvas)

Четыре панели вверху:

| Панель | Содержимое |
|--------|------------|
| СЧЁТ | Текущие очки |
| РЕЙТИНГ | Live-место (#1, #2, …) |
| КОМБО·ПРЫЖ | Множитель комбо + оставшиеся прыжки |
| ЖИЗНИ | ♥♥♥ / ♥♥♡ / ♥♡♡ |

Стиль панелей: glassmorphism (полупрозрачный фон, blur, тонкая красная обводка).

### 9.3. Мобильные кнопки

- Внизу экрана: **◀ ▶** слева, **⇧** справа.
- Появляются только на touch-устройствах во время игры.
- Крупные, с подсветкой при нажатии.

### 9.4. Game Over

| Элемент | Содержимое |
|---------|------------|
| «GAME OVER» | Заголовок |
| Игрок | Имя |
| Счёт | Финальный результат |
| Рекорд | Лучший личный |
| Место | 🥇 / 🥈 / 🥉 / #N |
| Топ‑10 | Таблица с подсветкой текущего результата |
| «ЗАНОВО» | Рестарт без возврата в меню |

## 10. Визуальный стиль (текущее состояние)

### Палитра

| Роль | Цвет | Hex |
|------|------|-----|
| Фон (верх) | Почти чёрный | `#030308` |
| Фон (низ) | Тёмно-синий | `#141432` |
| Spider-Man красный | Акцент | `#e63946` |
| Spider-Man синий | Акcent | `#1a3b8e` |
| Бирюза (рейтинг, glow) | Акцент | `#00cec9` |
| Золото (комбо, звёзды) | Акцент | `#fdcb6e` |
| Розовый (подушки) | Акцент | `#fd79a8` |
| Платформы | 6 неоновых оттенков | фиолетовый, жёлтый, бирюза и др. |

### Фон (canvas)

- Градиент ночного неба.
- **Parallax-силуэты города** (3 слоя, окна с жёлтым светом).
- Мелкие **звёзды**.
- Едва заметный **паутинный паттерн** (красные дуги).

### UI (HTML/CSS)

- Шрифты: **Bebas Neue** (заголовки), **Inter** (текст).
- Тёмная тема, glassmorphism-панели.
- Градиентный текст на счёте (синий → красный).
- Кнопки с hover/active-состояниями, красная обводка на активной сложности.

### Juice-эффекты

- Частицы при прыжке, сборе, падении.
- Screen shake при ударе / обрушении платформы.
- Всплывающий текст при комбо.
- Squash платформ при приземлении.
- Процедурные звуки (Web Audio API): прыжок, сбор, комбо, удар, game over.

## 11. Техническая информация

| Параметр | Значение |
|----------|----------|
| Стек | Vanilla JS, HTML5 Canvas, CSS3 |
| Без фреймворков | Нет React, Phaser и т.п. |
| Game loop | Fixed timestep 60 Hz |
| Рендер | Canvas 2D, DPR-aware (Retina) |
| Звук | Web Audio API (oscillator) |
| Хранение | localStorage |
| Деплой | GitHub Pages |
| Макс. ширина | 800 px (центрированный контейнер) |
| Ориентация | Портрет и ландшафт (адаптив) |

### Структура кода

```
js/config.js    — константы, палитра, сложность
js/math.js      — векторы, коллизии, утилиты
js/storage.js   — рейтинг, подпись счёта
js/input.js     — клавиатура, тач, кнопки
js/entities.js  — игрок, платформы, бонусы
js/systems.js   — частицы, звук, фон, спавнер
js/game.js      — ядро, state machine, loop
index.html      — разметка экранов
style.css       — UI и экраны
```

## 15. Зоны для улучшения дизайна (на проверку)

> Этот раздел — для дизайнера / ревьюера. Текущая версия функциональна, но визуал может требовать доработки.

| Область | Текущее состояние | Возможное направление |
|---------|-------------------|----------------------|
| **Экран старта** | Много элементов в одном скролле | Упростить иерархию, вынести рейтинг на отдельный экран |
